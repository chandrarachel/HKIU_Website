CW3: JavaScript

  1. Event handler setup:
    • Event handler by JavaScript:
      1. Line 8 in program.js (DOM level 0 event handler). This event handler chooses a random promo_msg as the initial message everytime the window is loaded.
      2. Line 59, 62 in apply.js (DOM level 0 event handler). This event handler detects if the mouse enters or leaves the element "CoE2", respectively. It will change "CoE2" background color intorgb(221, 221, 221) when the user's mouse hovers above it and change it back to white when it is not hovered.
      3. Line 49 in program.js (DOM level 2 event handlers). When the video ended, this event handler will call the function loop_vid, which will change to the other video everytime the other current video ended.
      4. Line 77 in visit.js (DOM level 2 event handlers). This event will call the function validate_form when the 'submit' is received by clicking the submit button. The function will check the input from the form and displays an error if the input is invalid.
      5. Line 10, 19, 28, 37 in apply.js (DOM level 2 event handlers). When the user clicked one of the college headings, the corresponding part will be displayed and the other two colleges will be hidden.
      6. Line 67, 70, 75, 78, 83, 86, 93, 96, 101, 104, 109, 112, 119, 122, 127, 130, 135, 138 in apply.js: When the user hovers above one the the majors, its background color will change into gray. It will return to white when the mouse is not hovering above it.
      7. Line 225 in apply.js: When the user clicks the submit button, the functioon submitButton will be called. It will check the content inside the table and display error message if the content is invalid.
      8. Line 300 in apply.js: Calls the function clearButton when the clear button is clicked. This function clears out resets the table so the user can start over again. It will also reset the total number into zero.
      9. Line 324 in apply.js: This will call the function getRank when any of the rank of choice buttons is clicked. The function will check if the input is valid.
    • Event handler by HTML:
      1. Line 47 in apply.html and the JavaScript is on line 59, 62 in apply.js
  2. ES6 features:
    • let and const:
      1. let
        • Line 19 in program.js: let is used to declare a block-scoped local variable. This is used to ensure that the value can only be used in the block where the let variable is placed, so any modification of the variable "position" will only affect that block. Any other "position" outside of the block will not be affected.
        • Line 243, 273, 304, 323 in apply.js: let is used inside for loop to declare an initial value of the variable "i". In this case, it is important to use let so the changes of values of "i" in this block will not affect the "i" variables outside of the block.
      2. const
        • Line 1 in visit.js: const is used for the variable "form", which get the element "bookInformation". const is used because the value inside the variable will not change.
        • Line 67 in visit.js: const is used for the variable "reserveInput", since the value will not change. So, const is used since the variable do not need to be reassigned.
        • Line 1 in program.js: const is used to store the promo messages inside the array, since the value will remain constant.
        • Line 9, 16 in program.js: const is used for the variables which get the element "col1", since the value will remain constant.
        • Line 17 in program.js: const is used for the variable which selects the text content of promo_info, since it will stay the same.
        • Line 1, 2, 4, 5, 7, 8, 50, 58, 66, 74, 82, 92, 100, 108, 118, 126, 134, 151, 152, 153, 181, 200, 209, 224, 227, 299, 305, 308, 322 in apply.js: const is used to select or get elements from apply.html. Since the value inside the variables (the selected elements) will remain the same, const is used to declare these variables.
        • Line 212-217, 241, 242, 247 in program.js: These variables will always have the same value.
        • Line 248 in program.js: In this part, the for loop iterates over the elements of rankChecker.
        • Line 270 in program.js: The remaining length will remain constant (will not be reassigned).

      • Line 3 in apply.css (welcome banner) and line 193 in apply.css (table)

    • Sticky positioning:
      • Line 6 and 18 in navStyle.css (navigation bar stays on top when scrolled)
    • Absolute positioning:
      • Line 62 in homepage.css ("Welcome to HKIU" banner)
      • Line 56 in visit.css (set the position of the animation (pink sliding background when hovering above the title of booking information form))
    • Others:
      • Line 49 in homepage.css (positioning the background image of the welcome banner to the center)
  2. Different kinds of CSS3 techniques:
    • Border: Line 24 and 25 in about.css (left and right border of the welcome paragraph)
    • Round corner: Line 17 in visit.css (map), line 19, 84,100 in homepage.css (map, button, and border of the location section), line 26, 71, and 90 in apply.css (buttons, headings for the majors)
    • Gradient: Line 37 in mainStyle.css (gradient text animation in header), line 56 in homepage.css (welcome banner), line 4 in apply.css (welcome banner background)
    • Transform: Line 50-54 in navStyle.css (transform rotation for shaking text animation when user hovers above the navigation bar links)
    • Animation: Line 46 in mainStyle.css (control the animation of header text moving gradient, line 62 in navStyle.css (control animation of shaking navigation bar links when hovered))
    • Transition: Line 61 in visit.css (control the movement of animation when the title of the booking information form is hovered (pink sliding animation behind title))
  3. How styles are grouped:
    • Each part of the webpage are organized into different div according to its content. They are all named with a class name or an id name. An example is #location in visit.css that contains the HKIU map and location information
    • Forms for different colleges in apply.css are organized with different div; each with different names and ids. Thus, it is possible to make them appear when hovered and disappeared when they are not hovered
    • The styles are grouped into external stylesheets in a folder named "CSS". Each stylesheet is linked to the corresponding html pages with the link tag
